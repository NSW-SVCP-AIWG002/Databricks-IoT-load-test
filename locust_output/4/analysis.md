## 課題・備考

- **disconnect 完全解消**：keepalive=100秒 により、5分待機中に 100秒・200秒の 2回 PINGREQ が送られ、IoT Hub の切断タイムアウト（keepalive×1.5=150秒）を確実にカバーできるようになった。
- **RPS 47.72**：5分間隔（平均300秒）の理論値 68.3 req/s に対して低いのは ramp-up 期間の影響。定常状態での実 RPS は別途計測が必要。
- **Event Hub Throttling**：5分間隔（約68 req/s）は Run 2 の 90秒間隔（約228 req/s）の約 30% の負荷であり、スロットリングは大幅に軽減された見込み。
- **keepalive=100秒 の副作用**：PINGREQ の送信頻度が高くなるため IoT Hub への制御トラフィックが増加するが、実用上の問題にはならない範囲と推定。
